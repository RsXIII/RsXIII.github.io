Απαιτήσεις (κατά Davis)

Αν μια εταιρία επιθυμεί να συνάψει σύμβαση για ένα μεγάλο έργο ανάπτυξης λογισμικού, πρέπει να ορίσει τις ανάγκες της με έναν αρκετά αφηρημένο τρόπο ώστε η λύση να μην είναι προκαθορισμένη.

Οι απαιτήσεις πρέπει να γραφούν έτσι ώστε να επιτρέψουν σε πολλούς εργολάβους να κάνουν προσφορά για τη σύμβαση, προσφέροντας, ίσως, διαφορετικούς τρόπους ικανοποίησης των αναγκών της εταιρείας-πελάτη.

Αφού ανατεθεί η σύμβαση σε κάποιον εργολάβο, εκείνος πρέπει να γράψει έναν ορισμό συστήματος για τον πελάτη με περισσότερες λεπτομέρειες, ώστε ο πελάτης να μπορεί να καταλάβει και να επαληθεύσει τι θα κάνει το λογισμικό. Και τα δύο αυτά έγγραφα μπορεί να ονομαστούν έγγραφα απαιτήσεων για το σύστημα.
